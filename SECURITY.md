# 🔐 Security Policy

## 📌 Supported Versions

The following versions of **featuremind** are currently supported with security updates:

| Version   | Supported |
|----------|----------|
| 3.1.x    | ✅ Yes   |
| 3.0.x    | ⚠️ Limited support |
| < 3.0    | ❌ No    |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in **featuremind**, please report it responsibly.

📩 Contact:  
- Email: nivedithar127@gmail.com  


---

## ⏱️ Response Time

- Initial response: **within 48 hours**
- Status update: **within 3–5 days**
- Fix / patch timeline: depends on severity

---

## 🔍 What to include in your report

To help resolve issues quickly, please include:

- Description of the vulnerability  
- Steps to reproduce  
- Affected version(s)  
- Any proof-of-concept (if available)  

---

## ⚠️ Responsible Disclosure

Please **do not publicly disclose vulnerabilities** until a fix has been released.

We appreciate your effort in helping keep **featuremind** safe 🙏

---

## 🛡️ Scope

This policy applies to:

- featuremind Python library  
- GitHub repository  
- Released versions on PyPI  

---

## 💡 Notes

featuremind focuses on **data integrity and model reliability**.  
While it helps detect risks like data leakage, it is **not a security tool** and should be used alongside standard security practices.
